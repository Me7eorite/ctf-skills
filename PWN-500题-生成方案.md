# PWN 500 题生成方案

> 用法：每个批次的「考点关键字 / 生成策略 / 参考链接」可直接粘贴进 Hermes 的对应输入框。
> 参考链接以**完整 writeup** 为主，确保 research 阶段能凑齐「定义→特征→利用步骤→交叉验证」四要素。
> 末尾附 **PWN 出题/解题环境必装软件清单**。

## 三、通用生成策略片段（拼到每批策略开头）

```
deployment=container（pwn 题需起服务）；
docker-compose 只一个 service，端口 9000-9999。
每题提供 binary（+libc/ld 如需），flag 通过交互式 exploit 获取，禁止把 flag 明文写进二进制。
exp.py 必须用 pwntools 远程交互稳定打通（process 本地 + remote 远程一致）。
研究须产出并写入 sources[]（每条结论≥1个一级来源）：
1) 漏洞原语定义与触发条件；
2) 二进制中的可识别特征；
3) 完整利用链步骤（leak→控制流→getshell）；
4) ≥1篇公开 writeup 交叉验证；
5) 对应 glibc/内核版本差异与缓解（NX/Canary/PIE/RELRO/seccomp）。
缺任一项视为研究结论不足，需补检索后再进入设计。
```

## 四、各批次方案

### 批次 1 — 栈溢出基础（60 题；简24/中28/难8）
**考点关键字**
```
ret2win stack buffer overflow return address overwrite
ret2text shellcode disable stack canary leak
stack canary bypass via format leak
partial overwrite return address PIE
```
**生成策略**
```
难度配额：简24/中28/难8。易题关闭 PIE/Canary 的纯 ret2win；中题需泄露 canary 或 ret2text；难题叠加 PIE 部分覆盖。
栈布局每题须有差异（缓冲区大小/保护组合 NX·Canary·PIE·RELRO），同一组合最多 10 题。
exp.py 用 pwntools cyclic 定位偏移，禁止硬编码 flag。
```
**参考链接（writeup 为主）**
```
https://ir0nstone.gitbook.io/notes/types/stack/return-oriented-programming
```

## 五、PWN 必装软件清单（Ubuntu）

### 基础工具链
```bash
apt-get update && apt-get install -y \
  build-essential gcc-multilib g++-multilib gdb gdb-multiarch \
  binutils file xxd patchelf elfutils \
  python3 python3-pip python3-venv git wget curl ruby socat
```

### Python / pwntools 生态
```bash
pip3 install pwntools ROPgadget ropper LibcSearcher angr capstone unicorn
```

### GDB 增强（三选一或都装，按题切换）
```bash
# pwndbg（推荐 heap/pwn）
git clone https://github.com/pwndbg/pwndbg && cd pwndbg && ./setup.sh
# 或 GEF
bash -c "$(curl -fsSL https://gef.blah.cat/sh)"
# 或 peda
git clone https://github.com/longld/peda ~/peda && echo "source ~/peda/peda.py" >> ~/.gdbinit
```

### 利用辅助工具
```bash
gem install one_gadget seccomp-tools          # one_gadget / seccomp dump
pip3 install pwninit                           # 自动 patchelf+下载 libc（或用 cargo）
# libc 管理
git clone https://github.com/matrix1001/glibc-all-in-one   # 多版本 glibc
git clone https://github.com/niklasb/libc-database          # libc 偏移数据库
```

### 内核 / 多架构 pwn
```bash
apt-get install -y qemu-system-x86 qemu-system-arm qemu-system-mips qemu-user qemu-user-static \
  gcc-aarch64-linux-gnu gcc-arm-linux-gnueabihf \
  gcc-mips-linux-gnu gcc-mipsel-linux-gnu gcc-riscv64-linux-gnu \
  cpio bzip2 libelf-dev   # initramfs 打/解包
```

### 静态分析（出题验证）
```bash
# Ghidra（需 JDK；JDK 用 SDKMAN 安装）
curl -s "https://get.sdkman.io" | bash && source ~/.sdkman/bin/sdkman-init.sh && sdk install java 17.0.11-tem
# 下载 Ghidra release；IDA 视授权另装
apt-get install -y radare2
```

### 部署校验项（对应 build_status / solve_status）
- 每题 `checksec` 记录保护（NX/Canary/PIE/RELRO/FORTIFY）。
- 容器内 `socat` 暴露端口，`exp.py` 同时跑通 `process()` 本地与 `remote()` 远程。
- 内核题在 qemu 中跑 `run.sh` 验证可提权读 flag。
