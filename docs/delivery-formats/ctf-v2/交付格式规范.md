# CTF题目交付格式规范 v2.0

**适用范围**：所有CTF题目的打包交付，确保格式统一、内容完整。
**参考样板**：`docs/delivery-formats/ctf-v2/资源包/`（即下表所示目录结构的真实样例）

---

## 一、交付包整体结构

```
资源包/
├── 工具/                                 # 每题：writeup + exp 解题工具包
│   └── js-{类型}-{题目}exp.zip           # 内含 wp（题解源稿/脚本）和 exp 脚本
│
├── 题库资源/
│   ├── ctf-overview.xlsx                # 题目数据总表（每行 = 一道题）
│   └── deploy/                          # 题目部署交付目录
│       ├── js-{类型}-{题目}.zip         # 每题：docker 配置包（src/_files/Dockerfile/compose）
│       ├── enclosure/                   # 每题：选手附件
│       │   └── js-{类型}-{题目}.zip
│       └── report/                      # 每题：PDF 题解
│           └── js-{类型}-{题目}.pdf
│
└── 虚拟机资源/
    ├── docker-tar/                      # 仅 pwn / web（及其它容器题）的镜像 tar
    │   └── {题目}[端口]-{YYYYMMDD}.tar
    └── 镜像模板.xlsx                    # 记录上面 docker-tar 中所有 tar 文件的清单
```

> 说明
> - `工具/` 和 `题库资源/deploy/` 中**每道题各对应一个 zip**，文件名按下表统一命名。
> - `虚拟机资源/docker-tar/` 仅对 **存在 docker 镜像的题目**（pwn、web、以及其它需要容器的题）提供 tar 文件；纯附件题（crypto / reverse / stego / forensics / misc 等）不出 tar。
> - `镜像模板.xlsx` 必须与 `docker-tar/` 内的文件一一对应，新增/删除 tar 时同步更新。

## 四、各目录详细规范

### 4.1 解题工具包 `工具/js-{类型}-{题目}exp.zip`

**内含 writeup（wp）与 exp 两类文件，是一份完整的「可复现解题」包**。

```
js-{类型}-{题目}exp.zip
├── wp.md                       # 题解源稿（Markdown，**必须中文**，对应 report/ 下 PDF 的源文件）
├── exp.py                      # 主解题脚本，运行后应直接拿到 flag
├── exp2.py                     # 备用脚本（可选）
├── utils.py                    # 工具函数（可选）
├── requirements.txt            # 依赖清单（如适用）
└── [其他必要工具或样例输出]
```

要求：
- **wp + exp 必须放在同一个 zip 中**；report/ 下的 PDF 与本包内 `wp.md` 内容保持一致。
- **`wp.md` 正文必须使用中文撰写**（章节标题、步骤说明、分析全部使用中文；代码 / 命令 / 工具名保留英文原样）。`report/` 下的 PDF 必须与之中文内容一致。
- `exp.py` 能在指明的 Python 版本下直接运行并获取 flag。
- 不要在 EXP 包内放任何与解题无关的大体积资源。

### 4.2 Docker 配置包 `题库资源/deploy/js-{类型}-{题目}.zip`

仅 **需要部署容器的题目（web / pwn / 部分 misc / cloud 等）** 出该包。
内容是把题目源码与部署配置打成 zip：

```
js-{类型}-{题目}.zip
└── deploy/
    ├── src/                    # 题目源码
    ├── _files/                 # 启动脚本 / 配置（如 start.sh、nginx.conf）
    ├── Dockerfile
    └── docker-compose.yml      # 必须只有一个 service
```

要求：
- `docker-compose.yml` 必须**只定义一个 service**；数据库 / 缓存 / 队列等依赖必须装进同一个镜像或同一容器内启动。
- 该包能用于重新构建 `虚拟机资源/docker-tar/` 下对应的镜像 tar，构建结果与运行环境一致。
- 非容器题（crypto / reverse / stego / forensics / 多数 misc 等）**不需要**该包。

### 4.3 选手附件包 `题库资源/deploy/enclosure/js-{类型}-{题目}.zip`

提供给选手下载的附件。按题型决定是否出包：

| 题型 | 是否提供附件 | 附件内容 |
|------|-------------|----------|
| Crypto | ✅ | task.py、output.txt、公钥参数等 |
| Web | ❌ | 选手直接访问容器 |
| Pwn | ⭕ 可选 | binary、libc、ld（如需 |
| Reverse | ✅ | 二进制 / 脱壳样本 |
| Stego | ✅ | 图片 / 音频 / 视频载体 |
| Forensics | ✅ | 流量包 / 日志 / 磁盘 / 内存镜像 |
| Misc | ✅ | 压缩包 / 二维码 / 文本 |
| ICS | ✅ | PLC 程序、流量、配置 |
| AI | ✅ | 模型、样本、对抗样本 |
| Cloud | ⭕ 可选 | 云环境配置 |
| Mobile | ✅ | APK、证书、配置 |
| Blockchain | ✅ | 合约、交易、模拟私钥 |
| IoT | ✅ | 固件、设备配置、通信日志 |
| Auto | ✅ | CAN 日志、ECU 固件、协议数据 |

附件包内不得包含明文 flag、敏感凭据、生产数据。

### 4.4 PDF 题解 `题库资源/deploy/report/js-{类型}-{题目}.pdf`

格式：必须为 PDF，**正文必须使用中文撰写**（与 `工具/` 包内的 `wp.md` 中文内容一致；代码 / 命令 / 工具名保留英文）。

内容章节：

1. 题目分析（题型、考点）
2. 漏洞 / 弱点分析（技术细节）
3. 解题步骤（逐步可复现）
4. 工具使用（工具及参数）
5. Flag 获取（最终 flag 与获取过程）
6. 附录（可选：关键代码、截图）

质量底线：另一位测题人按本 PDF 即可独立解出 flag。

### 4.5 Docker 镜像 tar `虚拟机资源/docker-tar/{题目}[端口]-{YYYYMMDD}.tar`

只对 **web / pwn 及其它容器题** 出 tar。

约束：
- 文件名：`{题目}[端口]-{YYYYMMDD}.tar`
  - Web：`sqli_basic_001[8080]-20260520.tar`
  - Pwn：`stack_overflow_001[9999]-20260520.tar`
  - 其它容器题：`some_chal_001[10000]-20260520.tar`
- 镜像加载后的名称约定：`docker load` 之后镜像名为 `{名称}:{日期}`，例如 `sql/bool:202605`。
- tar 必须与 `题库资源/deploy/` 下同题的 docker 配置包构建产物一致。

端口规则：

| 题型 | 端口范围 | 是否必须标注端口 |
|------|----------|-----------------|
| Web | 80、8080、8000-8999 | ✅ |
| Pwn | 9000-9999 | ✅ |
| 其它容器题 | 10000-10999 | ✅ |
| 无容器题 | — | ❌（不出 tar） |

### 4.6 Excel 表格

#### 4.6.1 `题库资源/ctf-overview.xlsx`（题目总表）

每一行对应一道题，必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| 题目ID | 类型-序号 | crypto-001 |
| 题目名称 | 具体名称 | rsa_wiener_001 |
| 题目描述 | 简介 / 场景 | 某 RSA 系统使用小私钥加密… |
| 题型 | 所属类型 | Crypto |
| 难度 | Easy / Medium / Hard | Medium |
| 考点 | 核心考点 | Wiener 攻击 |
| 分值 | 题目分值 | 200 |
| flag格式 | flag 前缀 / 模板 | flag{wiener_…} |
| 状态 | 待验证 / 通过 | 通过 |

#### 4.6.2 `虚拟机资源/镜像模板.xlsx`（tar 文件清单）

记录 `docker-tar/` 下所有 tar 的元数据，新增 / 删除 tar 时同步：

| 字段 | 说明 |
|------|------|
| 题目名称 | 对应题目 |
| 镜像文件 | tar 包文件名 |
| 端口 | 服务端口 |
| 基础镜像 | ubuntu / python / nginx 等 |
| 启动命令 | 容器启动命令 |

## 五、交付检查清单

### 5.1 必检项（P0）

| 检查项 | 方式 | 失败处理 |
|--------|------|----------|
| `工具/` 下每题有 zip，且含 wp + exp | 解压抽查 | 补齐缺失文件 |
| `题库资源/deploy/` 下每个容器题有 zip | 列表比对 | 补齐 deploy 包 |
| `题库资源/deploy/enclosure/` 命名前缀正确 | 正则匹配 | 重命名 |
| `题库资源/deploy/report/` 下每题有 PDF | 目录对照 | 补 PDF |
| `docker-tar/` 下每个容器题有 tar | `docker load` 测试 | 重建镜像 |
| `镜像模板.xlsx` 与 `docker-tar/` 一一对应 | 列表比对 | 同步更新 |
| `ctf-overview.xlsx` 行数 == 题目数 | 数量比对 | 补行 |
| 运行 EXP 能拿到 flag | 跑 `exp.py` | 修脚本或修题 |

### 5.2 推荐项（P1）

| 检查项 | 说明 |
|--------|------|
| Excel 字段填写完整 | 无空必填字段 |
| 附件无敏感信息 | 无硬编码 flag、无真实凭据 |
| 报告步骤清晰 | 人工审核可复现 |
| `工具/` 中 `wp.md` 与 `report/` PDF 内容一致 | 抽样对比 |
| `wp.md` 与 PDF 正文为中文 | 打开检查首尾章节 |
