# add-failure-classification-and-backfill

Tracks openspec change: [`openspec/changes/add-failure-classification-and-backfill/`](openspec/changes/add-failure-classification-and-backfill/)

## Summary

把"研究 run 失败"从一行原始字符串升级成可操作的运维体验，并填上"已 failed 但日志完整"的恢复缺口。两件事共享底层解析逻辑，作为一组能力同时交付。

1. **失败原因分类（`research-failure-classification`）**——后端纯函数 `classify_last_error` 将 `last_error` 映射成闭枚举 `(category, title, description, actions)`；run DTO 增量加 5 个派生字段；进度卡红色 alert 重渲染为图标 + 中文标题 + 描述 + 推荐动作 + 可折叠原文；运行历史表加"失败原因"列。
2. **从日志恢复研究结果（`research-result-backfill`）**——新增 `ResearchBackfillService` + `POST /api/research/runs/{run_id}/backfill`，UI 弹窗 preview → confirm → apply 二步交互；CLI `challenge-factory research backfill --run-id ... --dry-run` 和 `--all-recoverable --apply` 用于审计与历史积压批处理。
3. **共用底层抽出**——把 `_parse_research_output` 拆为无 I/O 解析与显式 raw-text materialize；抽 `_persist_rescue_payload` 作为 lease-rescue 与手工 backfill 的共享落库 helper，避免漂移。

## Key design decisions

详见 [`design.md`](openspec/changes/add-failure-classification-and-backfill/design.md) 八条决策。摘要：

- 分类逻辑放后端（域知识 + CLI/metrics 复用）。
- `recoverable` 是低成本"候选标志"，仅检查 safe-log 边界 + 有序 stdout markers；真正资格由 preview 判定。
- backfill **只接受最新 `failed` run**：不允许手工接管 `running`，不允许恢复被 retry/active/completed sibling 取代的旧 attempt（`run_not_terminal` / `superseded_run` / `active_sibling_run`）。
- preview 与 apply 之间用日志 SHA-256 锁定字节边界，apply 在 parse **之前**校验摘要，不一致返 `preview_stale`。
- 安全日志统一边界：路径必须在 `paths.research_logs` 下、无逃逸 symlink、≤10 MiB、严格 UTF-8。
- 并发用 UUID 16 字节折叠出的稳定 signed-64 advisory lock，外加 `SELECT FOR UPDATE` 行锁（禁止 Python `hash()`）。
- 单条 apply **不在 CLI 暴露**——单条强制走 UI 的二次确认；CLI 只保留单条 `--dry-run` 与批处理 `--all-recoverable --apply`。

## Relationship to prior lease-rescue change

上一轮 `add-lease-rescue`（已上线）通过 `_try_rescue_from_log` 救回**还在 running**的过期 run。本次改动：
- **不替换**它，而是**复用**它的落库逻辑（共用 `_persist_rescue_payload`）。
- 把它的日志安全边界统一到与 manual backfill 同一个 safe-log helper，消除两条路径的边界漂移。
- 填补它够不到的"已 failed 但日志完整"的存量缺口（操作员手工 backfill 路径）。
