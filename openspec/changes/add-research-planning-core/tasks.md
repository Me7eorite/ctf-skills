      2. **Start a daemon thread** running `_heartbeat_loop(run.id, agent_id, run.claim_token, lease_seconds)` that calls `job_service.heartbeat(run.id, agent_id, run.claim_token, lease_seconds)` every 30 seconds until a `stop_event` is set. Each iteration opens its own short session (SQLAlchemy sessions are not thread-safe). Hermes may take 5–25 minutes; heartbeats run in parallel and can request cancellation.
      4. **Stop the heartbeat thread** (`stop_event.set()`, `thread.join(timeout=5)`).
      5. If `lost_lease` is set, terminate Hermes if still running, discard any output, and return without terminal persistence. Fencing in the job service prevents stale completion even if this guard is missed.
      6. Parse stdout JSON. On any error (non-zero exit, invalid JSON, missing fields, finding with empty `source_indices`): call `job_service.mark_run_failed(run.id, agent_id, run.claim_token, last_error=<reason>, log_path=...)`. Catch `StaleClaimError` and skip — our claim_token no longer matches a `running` row (lease expired during our Hermes invocation, `claim_next_run` recovery has already marked this row `failed`, and a sibling retry row may be in flight with a different owner). Do NOT call `mark_run_failed` because we no longer own the lease. Return.

- [ ] 8.4 Create `src/services/research_worker.py::ResearchWorker(paths, job_service, agent_executor)`. Public method `run(agent_id: str, *, loop: bool, max_jobs: int, poll_interval_seconds: float, lease_seconds: int, hermes_timeout_seconds: int) -> dict`. SHALL validate `0 < hermes_timeout_seconds < lease_seconds` at entry; if not, raise `ValueError` with a clear message before any DB call. Main loop:
      ```
      processed = 0
      while True:
          run = job_service.claim_next_run(agent_id, lease_seconds)
          if run is None:

      ```
      `KeyboardInterrupt` (or SIGINT / SIGTERM caught and converted) SHALL: (a) set `stop_event` so the heartbeat daemon thread exits at its next iteration; (b) terminate the Hermes subprocess (best-effort `Popen.terminate()` then `kill()` after a 5s grace) so the system call returns to the executor; (c) **NOT** call `mark_run_completed`, `mark_run_failed`, or `complete_run_with_results` — the currently-claimed run is intentionally left in `running` state. Its lease will expire naturally and `claim_next_run`'s recovery path on the next worker will mark it `failed` with `last_error='lease expired'` and chain a retry row if attempts remain. The method then returns `{"processed": processed, "agent_id": agent_id, "interrupted": True}`. This keeps the shutdown path symmetric with worker crashes: both produce the same DB end state (one stale `running` row that lease recovery handles), so operators do not need a distinct "graceful shutdown" code path.
- [ ] 8.5 None of `ResearchJobService`, `ResearchAgentExecutor`, `ResearchWorker` SHALL import from `web`. Module-level imports SHALL only include stdlib, third-party, `persistence`, `hermes`, `domain`, and `core`.
